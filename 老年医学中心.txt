患者数据中心 — 前端 H5 项目需求整理
根据图片中的需求表格，以下是结构化梳理结果：

一、项目概述
系统名称： 患者数据中心
端类型： H5（移动端/Web端）
核心设计原则： 以身份证号码作为系统全局唯一识别 ID 和数据索引

二、核心模块拆解
模块 1：数据采集
子模块	字段内容
就医数据
就诊科室、病种、诊疗记录、检查报告、用药情况、手术史、复诊频率
个人基础数据
年龄、性别、身份证号码、籍贯、联系方式、过敏史
主动填写数据
健康问卷、慢病登记、健康需求反馈
备注：

①非使用功能项，首置开发和对接模块
②身份证号码是系统全局唯一识别与索引 ID
模块 2：标签体系
标签类型	标签内容
基础标签
年龄、性别、学历
诊疗标签
病种、就诊科室
健康标签
慢病类型、健康风险等级、康复阶段、健康需求
推送标签
互动率、互动频率、标号频率、浏览偏好
诊疗阶段
门诊 / 检查 / 住院 / 手术 / 出院 / 体检
备注：

①内建标签数据字典体系（含标签 ID 映射集合等）
②标签数据字典可维护（可扩展、可修改、可删除）
③【二期建议】基于用户数据自动推理元标签（精确认状态）
模块 3：服务组群
功能： 入组管理 — 基于多种标签的入组筛选预处理，用于后续信息精准推送

详细要求：

支持多标签组合的筛选设置
用户根据筛选标签组合自动入组
入组后可持续维护（包括退出用户的增删改查）
服务组群可复用
组群维护后，仅对后续关联的推送任务生效，已推送任务不受影响
模块 4：用户 360 信息展示
功能： 全量服务信息展示（单用户视角）

详细要求：

以身份证号码作为全局数据索引唯一 ID
以时间轴倒序方式呈现历年推送消息信息
可查看服务情况及短信信息
所有数据统一结构化归类处理
不可修改任何结果数据（确保执行数据的真实性）
模块 5：信息检索
功能： 组合检索

详细要求：

支持基于用户基本信息的检索（姓名、手机号码、身份证号码、病种、就诊科室等）
支持模糊检索
支持组合检索
以身份证号码作为全局数据索引唯一 ID
所有数据统一结构化归类处理
不可修改任何结果数据
三、关键约束与设计原则
约束	说明
唯一标识
身份证号码 = 系统全局唯一 ID
数据不可篡改
结果数据只读，确保真实性
标签可维护
字典体系支持扩展、修改、删除
推送隔离
组群变更不影响历史已推送任务
二期功能
自动标签推理（暂不开发）
